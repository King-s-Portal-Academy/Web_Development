The project timeline is the beating heart of every project. It captures the essence of what the project will accomplish and how it will be done. A project timeline can be highly detailed, with hundreds of tasks and subtasks, or very simple, listing only a few deliverables and deadlines. Regardless of how it is constructed, a project timeline needs to capture the information such as the list of tasks to be completed, the dates on which the tasks need to be complete, the expected duration of each task, dependencies between tasks, etc.
